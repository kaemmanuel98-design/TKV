3. Agent IA Biblique (LangChain + RAG)
3.1 Architecture de l'Agent
L'agent utilise le pattern RAG (Retrieval-Augmented Generation) : avant de repondre, il cherche les passages les plus pertinents dans la base de connaissances vectorielle de TKV, puis les injecte dans le prompt envoyé a GPT-4o. Cela garantit des reponses ancrees dans les contenus de la plateforme.

L'agent ne repond pas 'de lui-meme' : il cite et s'appuie sur les livres, cours et podcasts de TKV. Cela renforce l'autorite de l'auteur et justifie le contenu exclusif.

4. Base de Connaissances Vectorielle (pgvector)
4.1 Principe
La base vectorielle est le coeur de l'intelligence de TKV. Chaque contenu (livre, chapitre, cours, episode de podcast, article) est decoupé en chunks de 500 tokens, convertis en vecteurs d'embedding (1536 dimensions avec text-embedding-3-small d'OpenAI), puis stockes dans PostgreSQL avec l'extension pgvector.
Lors d'une question, le systeme calcule la similarite cosinus entre le vecteur de la question et tous les chunks, et retourne les 5 plus proches. Ce mecanisme garantit des reponses pertinentes et sourcees.

4.2 Schema de la Table knowledge_chunks
  PostgreSQL + pgvector  

id (UUID, PRIMARY KEY)
content_id (UUID, FK vers la table contents)
content_type (TEXT : 'book' | 'course' | 'podcast' | 'article')
chunk_index (INTEGER : position du chunk dans le document)
chunk_text (TEXT : texte brut du chunk, 500 tokens max)
embedding (VECTOR(1536) : vecteur OpenAI text-embedding-3-small)
metadata (JSONB : titre, auteur, chapitre, langue, date de publication)
language (TEXT : 'fr' | 'en' | 'es')
created_at (TIMESTAMPTZ)

INDEX : CREATE INDEX ON knowledge_chunks USING ivfflat (embedding vector_cosine_ops) WITH (lists = 100);

5. Analyseur de Perspectives — Mediateur IA
5.1 Vision et Unicite
L'Analyseur de Perspectives est la fonctionnalite la plus innovante de TKV v3.0. Sur une question sensible (existence de Dieu, miracles, evolution vs creation, souffrance et foi...), l'IA agit comme un mediateur neutre : elle presente les arguments des croyants ET des sceptiques de maniere equilibree, puis propose une synthese intellectuellement honnete.

Aucune application chretienne sur le marche ne propose cela. C'est un argument de vente unique et puissant pour attirer les sceptiques et les curieux, qui representent 60% du public potentiel de TKV.

5.2 Fonctionnement
Phase
Description
Detection de la question
L'agent detecte si la question est 'controversee' (existence de Dieu, preuves historiques, miracles, souffrance...) et active automatiquement le module Analyseur
Recherche bi-perspectiviste
pgvector cherche des chunks pertinents dans 2 categoies : arguments 'pro-foi' ET arguments 'sceptiques / scientifiques'
Structuration GPT-4o
GPT-4o produit une reponse en 3 parties : (1) Perspective croyante, (2) Perspective sceptique, (3) Synthese equilibree de l'auteur TKV
Affichage dans l'app
Carte interactive avec 3 onglets : Croyants / Sceptiques / Synthese. L'utilisateur navigue librement.
Invitation communautaire
L'utilisateur peut poster l'analyse dans le fil communautaire pour continuer le debat avec d'autres membres

5.4 Regles de Neutralite (Guardrails)
L'IA ne prend jamais parti au-dela des contenus de l'auteur TKV
La partie 'Synthese' reflète toujours la vision de l'auteur, clairement identifiee
Aucun contenu politique ou partisan ne peut etre genere
Un avertissement clair indique que l'analyse est produite par une IA
L'utilisateur peut signaler une analyse incorrecte ou biaisee
